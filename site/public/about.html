OpenPOIs

An open-source toolkit for comparing and conflating Points of Interest (POIs) across major geospatial datasets.

What is this?

OpenPOIs downloads current US-wide POI snapshots from multiple publicly available sources — currently OpenStreetMap and Overture Maps — and conflates them into a single unified dataset. The web map lets you explore each source side by side, or view the conflated output with a confidence score showing how well each POI is corroborated across sources.

The project is also a Python library for modeling POI stability over time using historical OpenStreetMap data. It fits an empirical Bayes Poisson model to estimate per-category change rates, giving a sense of how likely a given POI is to have changed since it was last observed.

Data Sources

OpenStreetMap

~7.8M

US POIs from the Geofabrik extract, filtered to amenity, shop, leisure, and healthcare tags.

Overture Maps

~7.2M

Places from the Overture Maps Foundation dataset, queried directly from their public S3 bucket via DuckDB.

All snapshots are US-wide and are periodically refreshed. The release date for each snapshot is tracked in the project configuration.

The Conflation Process

Conflation matches POIs across the three sources into a single unified record. The process works as follows:

Taxonomy normalization: Each source uses its own category system. OpenPOIs maps all three to a shared set of ~50 labels (e.g., Cafe, Bank, Pharmacy) using manually curated crosswalks. This shared taxonomy is used both for filtering in the web map and to set per-category match distances. See the full taxonomy reference for a complete mapping.

Spatial matching: Within each shared-label group, nearby POIs from different sources are candidate matches. Match radii vary by category — a small neighborhood coffee shop uses a tighter radius than a large hospital campus.

Confidence scoring: Each conflated POI receives a confidence score from 0 to 1, representing the estimated probability that the POI is real and currently present.

The confidence score is visualized on the map with a color ramp:

Level Score Description
Low < 30% Typically included in a single data source with low confidence
Medium 30% – 70% Typically included in multiple data sources with low confidence, or a single data source with medium-high confidence
High 70% – 90% Typically included in multiple data sources with medium confidence, or a single data source with high confidence
Very high > 90% Typically included in multiple data sources with high confidence

Data Access & Licensing

OpenStreetMap data is distributed under the Open Database License (ODbL). Because the conflated dataset incorporates OpenStreetMap, the combination is also covered by ODbL. You are free to copy, distribute, and use the data, provided that you attribute OpenStreetMap contributors and share any derivative databases under the same license.

The raw snapshot and conflated Parquet files are publicly available on Amazon S3 and can be queried directly with DuckDB, boto3, or any tool that supports S3 access.
